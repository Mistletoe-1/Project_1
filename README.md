# 澄序零售运营后台

> 现代化零售运营管理中台，为零售企业提供一站式运营管理解决方案。

## 项目概述

**澄序零售运营后台** 是一款基于 React 构建的现代化零售运营管理系统，专注于为零售企业提供高效、直观的运营管理工具。系统包含五大核心模块，覆盖订单管理、商品库存、会员运营、活动策划等关键业务场景。

### 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| React | 18.3.1 | UI 框架 |
| React DOM | 18.3.1 | React DOM 渲染 |
| Vite | 5.4.11 | 构建工具 |
| @vitejs/plugin-react | 4.3.4 | Vite React 插件 |
| Playwright | 1.61.1 | 端到端测试框架（预留） |

### 设计特点

- **现代化 UI**：采用清新自然的绿色系配色方案，搭配柔和的渐变背景
- **响应式布局**：完美适配桌面端、平板和移动端设备
- **流畅交互**：丰富的微动画和过渡效果，提升操作体验
- **无障碍设计**：支持 ARIA 标签和键盘导航

## 功能模块

### 1. 运营看板 (Dashboard)

核心数据概览页面，提供关键业务指标的实时监控。

**功能特性**：
- **核心指标卡片**：今日成交额、待处理订单、新客转化率、库存健康度
- **销售趋势图**：近 7 日成交表现折线图，支持查看详细报表
- **渠道结构分析**：各销售渠道贡献占比可视化
- **今日待办**：运营事项清单，支持任务完成状态切换
- **运营建议**：智能运营策略推荐

### 2. 订单中心 (Orders)

订单全生命周期管理，支持订单状态流转和风险监控。

**功能特性**：
- **订单列表**：展示订单号、客户、城市、金额、渠道、状态等信息
- **状态筛选**：支持按全部、待审核、待发货、运输中、售后中、已完成筛选
- **搜索功能**：支持按订单号、客户名、城市、渠道搜索
- **订单处理**：一键推进订单到下一状态节点
- **风险标签**：标记订单风险等级（普通、高价值、需跟进）

### 3. 商品库存 (Products)

商品信息管理与库存监控，支持快速补货操作。

**功能特性**：
- **商品卡片**：展示商品 ID、名称、分类、库存、销量、售价
- **库存预警**：低库存商品高亮显示，库存进度条可视化
- **快速补货**：一键增加 24 件库存，自动生成补货记录
- **搜索功能**：支持按商品名称、分类、ID 搜索

### 4. 会员运营 (Members)

会员分层管理与复购分析，助力精准营销。

**功能特性**：
- **会员统计**：活跃会员数、复购率、客单价核心指标
- **会员列表**：展示会员姓名、等级、消费金额、活跃度
- **会员标签**：高复购、价格敏感、新品偏好、待唤醒等标签
- **活跃度可视化**：会员活跃度进度条展示
- **搜索功能**：支持按会员名、等级、标签搜索

### 5. 活动计划 (Campaigns)

营销活动编排与预算管理，支持活动全流程管理。

**功能特性**：
- **新建活动**：填写活动名称、负责人、投放渠道、时间范围、预算等
- **活动列表**：展示活动状态、名称、负责人、日期、预算占用
- **预算可视化**：环形进度图展示活动预算占用比例
- **状态管理**：排期中、进行中、复盘中三种状态

## 快速开始

### 环境要求

- Node.js >= 16.0.0
- npm >= 8.0.0

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

开发服务器将在 `http://127.0.0.1:5174` 启动。

### 构建生产版本

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

### 预览生产版本

```bash
npm run preview
```

## 核心功能详解

### 通知系统

系统内置消息通知功能，支持：
- 实时消息展示（订单待审核、低库存提醒、活动复盘）
- 未读消息计数
- 一键标记全部已读
- 单条消息标记处理

### 数据模拟

项目使用 `mockData.js` 提供完整的模拟数据，包含：

- **导航菜单**：5 个功能模块配置
- **核心指标**：4 项关键业务数据
- **销售趋势**：7 日成交数据
- **渠道数据**：4 个销售渠道贡献
- **订单数据**：6 条订单记录
- **商品数据**：5 件商品信息
- **会员数据**：4 位会员信息
- **活动数据**：3 个营销活动
- **待办任务**：4 项运营任务

### 状态管理

应用使用 React Hooks 进行状态管理：

| 状态 | 说明 |
|------|------|
| `activeView` | 当前激活的视图模块 |
| `search` | 全局搜索关键词 |
| `orderFilter` | 订单状态筛选条件 |
| `orders` | 订单数据列表 |
| `products` | 商品数据列表 |
| `campaigns` | 活动数据列表 |
| `tasks` | 待办任务列表 |
| `notifications` | 通知消息列表 |
| `toast` | Toast 提示消息 |

## 响应式适配

系统支持多端设备适配：

- **桌面端（> 1120px）**：完整的双栏布局，侧边栏展开显示菜单文字
- **平板端（761px - 1120px）**：侧边栏收缩为图标模式，统计卡片和商品卡片变为两列
- **移动端（≤ 760px）**：侧边栏变为底部导航，布局变为单列，弹窗适配底部弹出

## 开发指南

### 添加新模块

1. 在 `src/data/mockData.js` 中添加新模块的导航配置和数据
2. 在 `src/components/` 目录下创建新的组件文件
3. 在 `src/App.jsx` 中导入新组件并添加路由逻辑

### 添加新图标

在 `src/components/Icons.jsx` 的 `paths` 对象中添加新图标的 SVG 路径数据。

### 自定义样式

在 `src/styles.css` 中修改 CSS 变量或添加新的样式类。
